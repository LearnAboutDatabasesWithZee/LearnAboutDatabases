Now that we have gone over a brief theory of Databases and if you're still here and curious about how to design your own database from scratch, the next few hours will be pivotal for your journey in data management.

Project: Learning Diary Database

In this practical, we will create a fully functional database system called Learning Diary. The system will be developed across three structured phases:

Phase 1: Database Logic

  • Define the key entities such as Diary Entries, Users, and Tags
  • Design attributes like entry_date, mood, topic, and reflections
  • Choose appropriate data types for each field (e.g., TEXT, DATE, BOOLEAN)
  • Apply relational constraints such as Primary Keys, Foreign Keys, and NOT NULL
  • Establish entity relationships: Users ↔ Diary Entries (one-to-many)

Phase 2: Data Operations (Data Ops)

  • Set up core Read, Create & Update operations using Microsoft Access VBA
  • Allow users to read past entries with filters (date, tags)
  • Implement forms to create new reflections
  • Enable update capability to revise entries
  • Offer search functionality with keywords and tag-based querying

Phase 3: ML Voice Assistant Extension

  • Incorporate a basic speech-to-text interface (using Vosk or Whisper)
  • Train a lightweight NLP model to recognize user intents like “Show today’s entry” or “Add new mood log”
  • Enable voice-triggered commands for creating, updating, or searching entries
  • Integrate a text-to-speech module for dynamic feedback

This modular approach helps reinforce your understanding of database theory and brings it to life with real-world interactivity and AI augmentation. Let's begin your Learning Diary journey!
