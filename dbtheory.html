Welcome to Learn About Databases With Zee - Theory Of Databases Page

Imagine we did not have databases, we most likely will be handwriting data and storing manually in books. And before we know it, there are bound to be problems which arise that are hard to work around, from these problems could be:

  1. How will we create backups of the data that we already have had to 'manually' record?
  2. Where will we store Data (including the backups) which has been recorded to preserve that from being accessed by unauthorised personnel and to prevent it from being stolen?
  3. And having said that, How could we check if the Data is still valid or indeed has become damaged?
  4. And it won't be too long after, when the Data begins to talk, what will we do then?

To address these issues there are ideas which may help:

  1. We need to find a way to organise the data initially in a structured way or put simple are required to design an Data organisation system.
  2. We need to store the data in multiple locations for backing up the Data.
  3. We need to speed up the rate at which we record data and create backups so we need to train people for specific tasks.
  4. We will need to safely store the data to prevent it from being stolen or accessed by unauthorised personnel so that Data integrity remains.
  5. We will need to have a system which can allow us to learn from the Data, key valuable insights to improve the life-cycle of the Data related operations.

What the fix is:

RDBMS - Relational Database Management System

An RDBMS is:

  1. A software system that stores, organises and thus manages data in structured formats known as tables.
  2. Each of the tables represents a set of related information, where rows are records and columns attributes
  3. In an RDBMS as Microsoft Access storing data in multiple locations for the purpose of backing up data is never an issue, there are a few options, we can manually save the Database file, there is an option to automate back up to a location of choice using VBA, and we can save the data on a network drive which is OneDrive.
  4. In as far as the need to train people for specific tasks to speed up the rate of recording data (as we continue to record, the data is saved and the backup is also updated), then there is the Split Database Architecture which is encouraged, where multiple users can use the Data which speeds up the rate at which data can be recorded or inputed.
  5. The split because the essential Data operations are categorised into two parts, Front end (forms, queries and reports) and Back end (tables/data).
  6. The safety of data is relevant to how the data resides in multiple locations, naturally speaking this precisely is data safety, the fact that is in more than one place, is therefore preserved. Now this is true so far as 'preventing the data from being stolen' issue is resolved however when considering what if unauthorised access was granted and how this could be avoided then we could reason with the Split Database Architecture once again which does facilitate for multiple users and thereby the Multi-User Login System (as programmable using VBA) which not only prevents access for personnel not authorised to view and use the Data but also governs the users actually using the Data in an organised fashion. As a final note on Data safety, for now, Data Integrity is the given name to the set of rules which are applied to Databases within the RDBMS which ensure that the Data is not damaged or put simple the values stored remain intact and are not changed because of these rules.
  7. Reporting allows for the creation of summaries of the Data collected which not only allows us to learn from the Data but also demonstrates how much more potential exists in the Data to be obtained and this compels to keep up with the Data operations cyclically continuing and in the meanwhile insight is gained and the quality of the Data improves,by studying the values and their relations with the rest of the values, most importantly the principal of the Data being recorded is maintained and the purpose of a Database remains.
