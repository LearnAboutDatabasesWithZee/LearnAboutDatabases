What Is Programming?

Programming is only a way of telling a computer what to do—step by step. It's like writing instructions for a robot or building a recipe.

🧁 Example: “Boil water, add tea bag, pour into cup” is a real-world algorithm. When written in code, it's programming!

Tutorial Objectives

  • Write and declare variables using the correct syntax
  • Discuss variables and how they relate to data types
  • Think critically about how data is stored and used in code

What Is a Variable?

A variable is similar to a labeled container or sticky note in your computer’s memory. You give it a name, assign a type, and store a value inside that you can reuse or change later.

🎒 Example: A college backpack has compartments labeled “Books,” “Lunch,” and “Keys.” Each acts like a variable: a name, a specific data type, and a value.

' Declare a string

' Declare an integer

' Declare a double
Dim assessmentScore As Double = 12.99

' Declare a boolean
Dim assessmentPassed As Boolean = True

Real-Life Analogy

Think of a variable as a luggage tag at the airport. The tag says "Owner: Mike". The word “Owner” is the label (variable name), and “Mike” is the data it stores.

How Programming Connects Data to What We See

Programming is more than just writing code—it’s the bridge between the data your application holds and what the user experiences. It takes that raw data and tells the computer what to do with it.

🎬 Let’s use a movie analogy:

  • The script is the data. It holds the dialogue, and instructions.
  • The director and crew are like your program. They interpret the script and bring it to life.
  • The finished movie is the output—what the audience sees on screen.

In software, it's the same:

  • Your data might be names, scores, settings, or options.
  • Your code processes that data and decides how the application behaves.
  • The output could be a message, calculation, visual element, or app behavior.

🎯 Without programming, data is like a movie script with no cast or camera. Code brings data to life and connects it to the user experience.

As we continue with the tutorials and move toward input/output, you’ll start seeing how code, variables, and data types work together to create interactive applications.
