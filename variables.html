What Is Programming?

Programming is only a way of telling a computer what to do—step by step. It's like writing instructions for a robot or building a recipe.

🧁 Example: “Boil water, add tea bag, pour into cup” is a real-world algorithm. When written in code, it's programming!

Tutorial Objectives

  • Write and declare variables using the correct syntax
  • Discuss variables and how they relate to data types
  • Think critically about how data is stored and used in code

What Is a Variable?

A variable is similar to a labeled container or sticky note in your computer’s memory. You give it a name, assign a type, and store a value inside that you can reuse or change later.

🎒 Example: A college backpack has compartments labeled “Books,” “Lunch,” and “Keys.” Each acts like a variable: a name, a specific data type, and a value.

Example Declarations

' Declare a string
Dim assessmentTitle As String = "Variables in Visual Basic"

' Declare an integer
Dim assessmentQuestions As Integer = 50

' Declare a double
Dim assessmentScore As Double = 12.99

' Declare a boolean
Dim assessmentPassed As Boolean = True

Real-Life Analogy

Think of a variable as a luggage tag at the airport. The tag says "Owner: Mike". The word “Owner” is the label (variable name), and “Mike” is the data it stores.

Frequently Asked Questions (FAQ)

Q: What is programming?

Programming is writing instructions a computer can follow. Like giving directions to a robot: "turn left", "grab item", "say hello".

Q: What is a variable?

A variable is a named space in memory that stores a value you can use or update in your program.

Q: What does Dim mean?

Abbreviated form of "Dimension", Dim is how you create a new variable in Visual Basic. It's like saying: “Make room for this named value.”

Q: Why are data types important?

They help your code treat each variable correctly—text, numbers, logic, or dates all need different handling rules.

Q: Can a variable change?

Yes! Most variables can be reassigned. For example, Dim correctAnswers As Integer = 20 could later become correctAnswers = 15.

Q: Why aren’t we using input/output yet?

We’re building foundational concepts first. Once you’re confident with variables and types, working with user input becomes much easier.

Q: How does programming use data?

Programming takes data (like numbers, text, or settings) and uses logic to give instructions to the application—telling it how to behave, respond, or process that data.
