Welcome to Conditions!

Now that you are familiar with variables and how to store information in a program, you're ready to explore how we make decisions with that information.

Why Do We Use Conditions?

Variables let us store facts and data. But what happens next depends on logic—and that’s where conditions are used.

Conditions let our programs ask questions like: “Is the score high enough to pass?” or “Are conditions used to make decisions?” Based on the answer (True or False), the program chooses what to do next.

Think of conditions as the brain of your code—choosing different paths, options, and behaviors depending on what’s inside the variables.

Real-Life Analogy: Airport Check-In

Imagine arriving at an airport. The staff checks your ticket:

  • If your seat is confirmed → you’re allowed to board.
  • If your seat is missing → you need to speak to a supervisor.

Your “ticket status” is like a variable. The program (airport staff) checks the status using a condition and acts accordingly.

One-Line Conditions

Sometimes you want to check a condition and respond quickly. In VB.NET, one-line conditions use this format:

If score >= 80 Then Console.WriteLine("Distinction")

This checks the value inside the variable score and runs the command if the condition is True. It's perfect for short, clear logic.

How Conditions Relate to Variables

Variables store your data. Conditions check those values and decide what to do. You’ll rarely write meaningful conditions without using variables!

Dim userAge As Integer = 16

If userAge >= 18 Then
    Console.WriteLine("You can vote.")
End If

Types of Conditional Statements

VB.NET gives you several ways to write conditional logic:

  • 1. If – Simple one-way check
    If condition Then action
  • 2. If...Else – Choose between two options
    If A Then... Else...
  • 3. If...ElseIf...Else – Handle multiple conditions
    If A Then... ElseIf B Then... Else...
  • 4. Select Case – Cleaner way to check many values
    Select Case variable... Case 10, Case 20...

Examples of Conditional Statements

1. If Statement

Used when you want to do something only if a condition is true.

Dim score As Integer = 90

If score >= 80 Then
    Console.WriteLine("Distinction!")
End If

2. If...Else Statement

Used when you want to choose between two outcomes.

Dim age As Integer = 16

If age >= 18 Then
    Console.WriteLine("Adult")
Else
    Console.WriteLine("Minor")
End If

3. If...ElseIf...Else Statement

Used when you need to check multiple possibilities in sequence.

Dim marks As Integer = 75

If marks >= 85 Then
    Console.WriteLine("A grade")
ElseIf marks >= 70 Then
    Console.WriteLine("B grade")
Else
    Console.WriteLine("Needs improvement")
End If

4. Select Case Statement

Best used when you're comparing one variable against multiple specific values.

Dim grade As Char = "B"c

Select Case grade
    Case "A"c
        Console.WriteLine("Excellent")
    Case "B"c
        Console.WriteLine("Good job")
    Case "C"c
        Console.WriteLine("Satisfactory")
    Case Else
        Console.WriteLine("Review needed")
End Select

Comparison and Logical Operators

Frequently Asked Questions (FAQ)

Q: What are the main types of condition statements?

If, If...Else, If...ElseIf...Else, and Select Case—each lets your program decide what to do based on what it knows.

Q: When should I use Select Case instead of If statements?

Use Select Case when checking a single variable against multiple possible values—it's cleaner and easier to read.

Q: Can I use variables inside conditions?

Absolutely. That's the whole point! We store data in variables, and then use conditions to decide what to do with that data.

Q: What if none of the conditions are true?

If you’ve included an Else or Case Else, that part runs. If not, the program skips that block entirely.

Q: Can I put one condition inside another?

Yes, that’s called nested logic. It lets you make complex decisions with multiple checks—just be sure to keep your indentation clean.

Q: How do I change the path based on a variable’s value?

You can track how many times a student has attempted an assessment using a variable—then apply a condition to restrict further attempts after a set number.

Dim attempts As Integer = 3

If attempts >= 3 Then
    Console.WriteLine("You have reached the maximum number of attempts for this assessment.")
Else
    Console.WriteLine("Attempt recorded. You have " & (3 - attempts) & " attempt(s) remaining.")
End If

This checks the number of attempts and shows different messages depending on the result. It's a practical example of using a variable and a condition together to control what a user is allowed to do next.
