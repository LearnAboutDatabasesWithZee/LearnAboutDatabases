📚 Learning Plan DB -Starter Project (Microsoft Access)

This system is designed as a Microsoft Access database project to document and track learning activities through structured sessions, workflows, and reflective audits. It blends task management with introspection—ideal for solo learners, personal development projects, or structured training programs.

Each session represents a unique learning event, branching into multiple workflow tasks with monitored progression. The schema encourages deliberate practice, symbolic notation, and iterative growth.

🧱 Entity Overview

TablePurpose
SessionDiaryLogCaptures session-level metadata and reflective notes
WorkflowBreaks down learning sessions into structured tasks
TaskAuditTracks progress states of each session-task pairing

🔗 Relational Dynamics

  • SessionDiaryLog anchors the learning record with LearningSessionID.
  • Workflow maps tasks to a session via LearningSessionID.
  • TaskAudit logs task progress, linking both LearningSessionID and WorkflowID.

🛠️ Step-by-Step: Build in Microsoft Access

  1. Create Database: Access → Blank DB → Name LearningSystem.accdb
  2. Create SessionDiaryLog:
    • LearningSessionID (AutoNumber, PK)
    • SessionStart, SessionEnd (Date/Time)
    • LearningSession (Short Text), Notes (Long Text)
  3. Create Workflow:
    • WorkflowID (AutoNumber, PK), LearningSessionID (Number, FK)
    • LearningTask (Short Text)
  4. Create TaskAudit:
    • LearningSessionID, WorkflowID (Number, FKs)
    • Progression – Lookup or Short Text
  5. Establish Relationships:
    • SessionDiaryLog → Workflow on LearningSessionID
    • SessionDiaryLog → TaskAudit on LearningSessionID
    • Workflow → TaskAudit on WorkflowID
    • Enable Referential Integrity
